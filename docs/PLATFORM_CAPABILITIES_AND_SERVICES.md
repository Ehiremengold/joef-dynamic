# School Management Platform
### Capabilities & Services Overview

A modern, web-based platform that helps schools run examinations, manage students and staff, track attendance, and produce professional, branded result documents — all from one secure portal, accessible on any device with a browser.

> **Use this document** as a leave-behind for school owners, principals, and administrators. Everything under **"Available today"** is live and working. Everything under **"On the roadmap"** can be built on request as part of a service engagement.

---

## 1. The problem we solve

Most schools still run on paper registers, hand-marked scripts, and manually typed result sheets. That means:

- Hours lost every term marking tests and re-typing scores into report cards.
- Arithmetic mistakes on results that damage the school's credibility with parents.
- Entrance exams that are hard to organise, mark, and rank fairly.
- Attendance registers that never make it onto the report card.
- No single, safe record of students, staff, and results — files get lost or damaged.

Our platform replaces that with a single online system that does the repetitive work automatically and keeps every record safe.

## 2. What the school can do

### Available today

**Student roster management**
- Add students with an admission number and full name; the system generates a secure 6-digit PIN for each student to log in with.
- Search and filter students by name and class.
- Reset a student's PIN, or deactivate/reactivate a student, at any time.
- Assign each student to a specific class (see multi-campus below).

**Staff accounts & access control**
- Create login accounts for teachers and administrators.
- Two clear roles: **Administrators** (full control of the school setup) and **Teachers** (day-to-day work like tests and attendance).
- Promote, demote, or deactivate staff. The system prevents the school from ever locking itself out (there must always be at least one active administrator).

**Multi-campus & flexible class structure**
- Run more than one campus from a single account (e.g. a main campus and a branch).
- Set up classes per campus, with flexible **arms/streams** — one campus can run several arms per level (e.g. SS2A, SS2B, SS2C) while another runs a single stream. Arms are added and named freely, not fixed.
- Assign a **class teacher** to each class.

**Subjects & curriculum**
- Maintain a central subject list for the school.
- Group senior-secondary subjects by department (Science / Art / Commercial) on one merged list.
- Choose which subjects each class offers.
- Mark subjects an individual student is **not** taking, so those subjects are correctly left off that student's records.

**Computer-Based Testing (CBT)**
- Create online tests for any class, with a full question bank (multiple-choice) per test.
- Set a duration per test; a server-based timer keeps everyone honest and keeps running even if a student closes the tab.
- Students take tests securely using their admission number and PIN.
- Answers are auto-saved as the student works — a dropped connection or flat battery never loses their progress; they can resume where they left off.
- Staff can grant extra time to a student when there was a genuine outage.

**Common Entrance examinations**
- Run entrance exams for prospective pupils who don't yet have a school account.
- Applicants register and sit the exam online.
- A single shared **access code** gates the entrance exam — staff activate it on exam day and switch it off afterwards.

**Automatic marking & results**
- Tests are marked instantly and accurately — no manual grading of objective questions.
- Every score is stored against the student and the exam.

**Professional, branded result documents (PDF)**
- **Result slips** — a clean, letter-headed slip for a single student's test, carrying the school's name, logo, and motto.
- **Student transcripts** — one student's full history of tests with their average.
- **Class result sheets** — every student's score for one exam, automatically ranked, with the class average.
- **Attendance on the result slip** — a student's attendance for the current term (days present out of total school days, as a percentage) is printed directly on their branded result slip.

**Attendance tracking**
- Class teachers record attendance per term as a simple cumulative total: the total number of school days for the class, and each student's days present.
- Fully editable at any time — nothing is locked, so corrections are easy.
- Only the assigned class teacher (or an administrator) can enter a class's attendance.
- Attendance flows straight through to the printed result slip.

**Academic sessions & terms**
- Set up academic sessions (e.g. 2025/2026) and terms (First / Second / Third).
- Mark the current term so attendance and results are always recorded against the right period.

**Security & data safety**
- Every record lives in a secure, cloud-hosted database with automatic backups.
- Strict access rules: sensitive data (like correct answers and results) is never exposed to students' browsers.
- Staff and students sign in through separate, protected login systems.
- Works on phones, tablets, and computers — no software to install.
